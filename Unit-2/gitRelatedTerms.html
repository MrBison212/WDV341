Git Related Terms

Version Control Software

It's a practice that helps track and manage changes to software code. It tracks any modification to the code within a database. If any mistake is made, the developer can turn around and compare older copies of code to newer copies to try to fix those mistakes.

Add

This Git command adds all of the files (new and modified) in the current directory to the index, getting them ready for the next git commit.

Commit

git commit is used to save your changes in the code to the local repository

Push

git push is used to upload local repository files to a remote repository. Pushing is how you transfer commits from your local repo to your remote repo.

Pull

git pull is essentially a combination of git fetch and git merge. git fetch updates all the remote tracking branches in a local repo, while git merge merges the corresponding remote tracking branch to the local working branch.

Clone

git clone is used to create a copy of a specific repository or branch within a repository. From what I've read, cloning a repository is typically only done once, at the beginning of the porject you're working on, and isn't usually cloned again.
